## Gestión de Recursos y Logística

### 1. Recursos Materiales

Para garantizar la continuidad del desarrollo y la estabilidad de la arquitectura Dockerizada.

* **Software:**
* **IDE:** Visual Studio Code (con extensiones para React Native y Docker).
* **Gestión de Base de Datos:** Prisma Studio para gestion de la **BBDD** a traves del modelo del **ORM**.
* **Pruebas de API-SOCKET:** Postman o Bruno.
* **SaaS/Cloud:** Firebase Console (Notificaciones), Expo Dev Tools.
* **Control de Versiones:** GitHub.


* **Hardware:**
* **Equipo de desarrollo:** Win11 - I7 12700H - 32GB RAM.
* **Servidor Local:** Debian 13 - I5 6500T - 16GB RAM .
* **Dispositivos de Prueba:** Xiaomi Redmi Note 11S, Nothing 3a.



### 2. Recursos Humanos

Al ser un proyecto hecho por una sola persona todos estos roles los he asumido yo:

| Rol | Responsabilidades Clave |
| --- | --- |
| **Analista / Product Owner** | Definición de requisitos de seguridad y lógica de flujos de mensajes. |
| **Desarrollador Fullstack** | Implementación de la API en Express, gestión de contenedores Docker y lógica de React Native. |
| **Especialista en Seguridad** | Implementación y auditoría de las librerías de cifrado P2P/E2EE. |
| **QA / Tester** | Pruebas de carga en WebSockets y validación de entrega de notificaciones en segundo plano. |
| **DevOps** | Configuración de despliegues en Docker y mantenimiento del servidor local. |

### 3. Asignación de Recursos en el Tiempo

Distribución estimada por fases de carga de trabajo:

1. **Fase de Inicio (20% del tiempo):**
* Fuerte presencia del **Analista** y **DevOps** para levantar la infraestructura Docker y el esquema de base de datos.


2. **Fase de Desarrollo Core (60% del tiempo):**
* Carga máxima del **Desarrollador Fullstack**. Se asignan los recursos de hardware para el desarrollo de la interfaz y la lógica de sockets.


3. **Fase de Seguridad e Implementación (10% del tiempo):**
* Intervención directa del **Especialista en Seguridad** para la fase de cifrado (actual fase del proyecto).


4. **Fase de Cierre y Pruebas (10% del tiempo):**
* Foco total del **Tester** y validación final en dispositivos físicos.
